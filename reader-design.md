# Reader 前端设计风格总结

## 1. 设计定位

Reader 当前主页面采用「Apple 风格极简阅读界面」路线，核心目标是：

- 减少视觉噪音，突出内容本身
- 用轻量动效提升“进入感”和“可点击感”
- 兼顾桌面端与移动端触控反馈

关键词：极简、留白、弱对比层次、玻璃质感、自然动效。

## 2. 视觉语言

### 2.1 色彩系统

- 主背景为纯白/浅灰语义变量，形成高可读性的阅读底板
- 文本使用深灰而非纯黑，降低硬对比带来的疲劳感
- 强调色使用 Apple Blue（accent）用于分类、交互提示
- 边框与分割线采用低存在感灰阶，强调结构而不抢内容

整体是“灰阶为主、蓝色点睛”的冷静风格。

### 2.2 排版系统

- 标题、正文、说明文案使用分级 typography 工具类
- 标题采用紧凑字距与较高权重，正文保持舒展行高
- 通过 `clamp` 实现流体字号，保证大屏与手机都保持比例协调
- 字体组合为衬线（品牌与大标题）+ 无衬线（信息文本）

观感上接近“编辑型杂志 + 产品化信息架构”。

### 2.3 空间与结构

- 页面采用大块留白和明确分段（Header / 主内容 / Footer）
- 内容区使用多列卡片网格（移动端到桌面端渐进增强）
- 年份分组作为视觉锚点，弱化装饰、强化时间结构

## 3. 卡片交互与微动效

## 3.1 进入动效

- 页面标题区使用淡入上移（fade + translateY）建立首屏节奏
- 卡片列表使用 `stagger` 逐项进入，形成“顺序阅读”的心理预期
- 单卡片使用弹簧参数（spring）从轻下移到位，显得有质感但不过度

## 3.2 悬停与触控反馈

- 桌面端：卡片 hover 轻微上浮，右上箭头同步位移与变色
- 移动端：卡片 tap 时缩放到 `0.98`，补足无 hover 场景下的按压反馈
- 删除按钮仅在 hover 组态中显现，避免常态界面干扰阅读

这套策略让交互反馈“可感知但不喧宾夺主”。

## 3.3 过渡节奏

- 统一使用中短时长（约 0.2s~0.6s）
- 缓动曲线以 ease-out / spring 为主，避免生硬线性动画
- 动画幅度控制在轻微位移和细小缩放，维持专业克制感

## 4. 组件气质与可用性

- 卡片容器使用 glass 风格（半透明背景 + 轻阴影 + blur）
- 元信息（分类、日期、阅读时长）采用低饱和信息层级，不压标题
- 标签采用小圆角胶囊样式，增加信息识别但不形成视觉主角
- 弹窗动效和背景毛玻璃一致，形成全局一致的交互语言

## 5. 当前风格结论

Reader 首页已经形成稳定且统一的设计系统：

- **视觉基调**：Apple 式克制、清洁、现代
- **信息组织**：以阅读流为核心，按年份分层
- **动效策略**：以“轻反馈 + 强可读”为准则
- **交互体验**：桌面与移动端均有明确点击反馈

适合作为后续页面扩展的母体风格。新增页面建议延续同样的：

- 语义色变量
- typography 分级体系
- 卡片化容器语言
- 轻量级微动画节奏
